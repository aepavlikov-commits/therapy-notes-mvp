# Заметки сессии — прототип (GigaChat)

Этот проект состоит из двух частей:
- `index.html` — сама страница приложения (фронтенд)
- `api/gigachat.js` — небольшой backend-сервер (serverless-функция), который безопасно общается с GigaChat

## Шаг 1. Получите сертификат Минцифры (обязательно, один раз)

Без этого файла backend не сможет подключиться к серверам GigaChat — это требование самого Сбера, не наша прихоть.

1. Откройте в браузере: https://gu-st.ru/content/lending/russian_trusted_root_ca_pem.crt
   (файл скачается на телефон/компьютер)
2. Переименуйте скачанный файл в `russian_trusted_root_ca.pem`
3. Положите этот файл в папку `api/` рядом с `gigachat.js` уууааауу

Если делаете это с телефона — это можно сделать прямо в GitHub (см. шаг 2), загрузив файл в папку `api` через веб-интерфейс GitHub, без компьютера.

## Шаг 2. Загрузите проект на GitHub

1. Зайдите на https://github.com и создайте бесплатный аккаунт, если его нет
2. Нажмите "New repository", дайте любое имя, например `therapy-notes-mvp`
3. Загрузите все файлы этого проекта через кнопку "Add file → Upload files" (это работает прямо с телефона в браузере, без приложений)
4. Не забудьте загрузить и сертификат `russian_trusted_root_ca.pem` в папку `api`

## Шаг 3. Подключите Vercel

1. Зайдите на https://vercel.com, зарегистрируйтесь (можно через GitHub-аккаунт — это самый быстрый способ)
2. Нажмите "Add New → Project"
3. Выберите ваш репозиторий `therapy-notes-mvp`
4. Нажмите "Deploy" — никаких настроек менять не нужно
5. Через минуту получите ссылку вида `https://therapy-notes-mvp.vercel.app`

## Шаг 4. Получите адрес вашего API

Ваш прокси будет доступен по адресу:
`https://therapy-notes-mvp.vercel.app/api/gigachat`

(подставьте свой реальный адрес сайта вместо `therapy-notes-mvp.vercel.app`)

## Шаг 5. Используйте в приложении

1. Откройте `https://therapy-notes-mvp.vercel.app` (главную страницу вашего сайта)
2. В поле "Подключение GigaChat" вставьте:
   - В первое поле — адрес вашего API из шага 4 (с `/api/gigachat` на конце)
   - Во второе поле — ваш Authorization key из личного кабинета GigaChat (developers.sber.ru)
3. Нажмите "Сохранить"
4. Дальше всё работает как в локальной версии: запись, транскрипция, генерация заметок

## Важно про безопасность

- Authorization key вводится только вами в своём браузере, не виден другим посетителям сайта
- Backend (`api/gigachat.js`) никогда не сохраняет и не логирует ваш ключ — он используется только для одного запроса и не сохраняется на сервере
- Каждый, кто откроет вашу ссылку без своего ключа, не сможет генерировать заметки — это нормально и безопасно

## Если что-то не работает

- Ошибка про сертификат → проверьте, что файл `russian_trusted_root_ca.pem` точно лежит в папке `api`, и название файла набрано без ошибок
- Ошибка 401/403 от GigaChat → проверьте, что Authorization key скопирован полностью, без пробелов
- Любая другая ошибка покажется прямо в интерфейсе с деталями — это поможет понять, что чинить дальше
Тестовая правка для пересборки.
